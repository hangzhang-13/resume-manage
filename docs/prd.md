# 需求文档

## 1. 应用概述

### 1.1 应用名称
面试简历管理系统

### 1.2 应用描述
本系统用于管理面试简历，用户上传简历后，系统智能提取关键信息并生成Excel表格，支持在线编辑功能。

## 2. 用户与使用场景

### 2.1 目标用户
人力资源部门工作人员、招聘负责人

### 2.2 核心使用场景
- 批量处理面试简历，快速提取关键信息
- 统一管理候选人信息，便于后续跟进
- 在线编辑和完善简历信息

## 3. 页面结构与功能说明

### 3.1 页面结构

```
面试简历管理系统
├── 简历上传页面
└── 简历管理页面
```

### 3.2 简历上传页面

#### 3.2.1 上传功能
- 用户选择本地简历文件进行上传
- 支持单个或多个简历文件上传
- 上传后系统自动进行信息提取

#### 3.2.2 上传反馈
- 显示上传进度
- 上传完成后提示成功或失败
- 提供跳转至简历管理页面的入口

### 3.3 简历管理页面

#### 3.3.1 表格展示
- 以Excel表格形式展示所有简历信息
- 表格列依次为：面试时间、用人部门、姓名、状态、性质、职位、年龄/工作经验、职级、工作履历、学历、面评、简历
- 每行对应一份简历的提取信息

#### 3.3.2 信息提取规则
- 系统从上传的简历中智能提取以下字段：
  - 面试时间
  - 用人部门
  - 姓名
  - 状态
  - 性质
  - 职位
  - 年龄/工作经验
  - 职级
  - 工作履历
  - 学历
  - 面评
  - 简历（原始简历文件）
- 若简历中无法提取某字段信息，则该单元格留空

#### 3.3.3 编辑功能
- 用户可点击表格单元格进行在线编辑
- 支持修改任意字段内容
- 编辑后自动保存

#### 3.3.4 Excel导出
- 提供导出功能，将当前表格数据导出为Excel文件
- 导出文件包含所有列和行信息

## 4. 业务规则与逻辑

### 4.1 简历提取逻辑
- 上传简历后，系统自动触发信息提取流程
- 提取完成后，在简历管理页面生成新行记录
- 姓名字段为必提取字段，其他字段可为空

### 4.2 数据存储
- 所有提取的简历信息需存储至后端数据库
- 原始简历文件需保存，并在表格中关联展示

### 4.3 编辑保存逻辑
- 用户编辑单元格内容后，系统自动保存至后端
- 编辑操作实时生效，无需手动提交

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 上传文件格式不支持 | 提示用户文件格式错误，拒绝上传 |
| 简历中无法提取姓名 | 提示提取失败，该简历不生成记录 |
| 简历中其他字段无法提取 | 对应单元格留空，允许用户后续手动编辑 |
| 编辑时输入内容为空 | 允许保存空值 |
| 导出Excel时无数据 | 提示当前无简历数据，无法导出 |

## 6. 验收标准

1. 用户在简历上传页面选择并上传一份简历文件
2. 系统完成上传并自动提取简历信息
3. 用户进入简历管理页面，看到新增一行记录，姓名字段已填充
4. 用户点击某个单元格进行编辑，修改内容后自动保存
5. 用户点击导出功能，成功下载包含所有简历信息的Excel文件
